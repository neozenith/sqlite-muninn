 The viz/ sub folder has a Makefile which describes the build system for what is a python backend (viz/server/) and a React frontend (viz/frontend/) AND we also have viz/frontend/public/wasm/ which is where a copy of the wasm build of sqlite-muninn extension gets built.

  The whole point of this sub folder project is to create a visual webapp to interact with demo databases under viz/frontend/public/demos/ as catalogued by viz/frontend/public/demos/manifest.json and they are built by the benchmarks/demo_builder and benchmarks/sessions_demo/ cli tools. 
                                                                                                                                                                                                                                                                                                 
  You can ignore the viz/frontend/public/demos/*.joblib files.                                                                                                                                                                                                                                   
                                                                                                                                                                                                                                                                                                 
  /escalators-not-stairs                                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                 
  The "escalator" critical function is:                                                                                                                                                                                                                                                          
  - WASM: http://localhost:5280/public/wasm/index.html                                                                                                                                                                                                                                           
  - Python Server Backed: http://localhost:5280/kg/query                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                 
  These are 2 parallel implementations of a similar interface to be able to query into each of these demo databases.                                                                                                                                                                             
                                                                                                                                                                                                                                                                                                 
  Each interface contains:                                                                                                                                                                                                                                                                       
  - search bar where I can enter search query                                                                                                                                                                                                                                                    
  - FTS Results that pull up chunks that match an FTS search of the query                                                                                                                                                                                                                        
  - Embedding results                                                                                                                                                                                                                                                                            
    - that uses an embedding model to embed the query and return results via the muninn hnsw embedding index,                                                                                                                                                                                    
    - which we then cross reference with the original chunk and list those chunks AND                                                                                                                                                                                                            
    - a 3D umap dimensionally reduced version of the reduced vector which we visualise using Deck.GL                                                                                                                                                                                             
    - hovering over a node in the Deck.GL vis shows hover text of the chunk text associated with the embedding                                                                                                                                                                                   
  - Knowledge Graph results                                                                                                                                                                                                                                                                      
    - Cytoscape.JS visualisation of the GraphRAG results returned                                                                                                                                                                                                                                
                                                                                                                                                                                                                                                                                                 
  The key thing is that there are 3 approaches to search being applied.                                                                                                                                                                                                                          
                                                                                                                                                                                                                                                                                                 
  The key distinction between the 2 parallel implementations:                                                                                                                                                                                                                                    
  - The python server backed implementation delegates sending the query to the python api's and it is behind the api that the query text gets converted into a vector embedding                                                                                                                  
  - The WASM implementation is ALL in browser memory, so the query text uses an embedding model in the browser via transformers.js to embed the query, then the frontend browser code (viz/frontend/public/wasm/script.js) uses the WASM version of sqlite and the muninn extension to perform   
  the queries and return the results for each of the visualisations.
